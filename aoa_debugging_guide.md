# AOA 디버깅 가이드 (USB 디버그 고려사항)

AOA(Android Open Accessory) 모드나 USB 호스트 기능을 사용하는 앱을 개발할 때는 안드로이드 기기의 USB 포트가 외부 하드웨어와 연결되어야 하므로, 일반적인 USB 케이블을 통한 ADB 디버깅이 불가능합니다. 이를 해결하기 위해 **네트워크(TCP/IP)**를 통한 디버깅 방법을 사용해야 합니다.

## 1. 네트워크를 통한 ADB 연결 방법 (TCP/IP)

USB 포트를 비워두고도 실시간 로그(logcat)를 확인하거나 앱을 설치하기 위해 다음 순서를 따르세요.

1.  **초기 연결**: 우선 안드로이드 기기를 USB 케이블로 컴퓨터에 연결합니다.
2.  **TCP 모드 활성화**: 터미널(명령 프롬프트)에서 다음 명령어를 입력하여 기기가 5555 포트로 네트워크 연결을 기다리도록 설정합니다.
    ```bash
    adb tcpip 5555
    ```
3.  **USB 케이블 제거**: 이제 안드로이드 기기에서 USB 케이블을 분리합니다.
4.  **IP 주소로 연결**: 안드로이드 기기와 컴퓨터가 **같은 와이파이(Wi-Fi)**에 연결되어 있는지 확인한 후, 기기의 IP 주소를 사용하여 연결합니다.
    ```bash
    adb connect <기기-IP-주소>:5555
    ```
    *(예: adb connect 192.168.0.10:5555)*
5.  **연결 확인**: 연결이 성공하면 이제 USB 연결 없이도 `adb logcat`이나 VS Code/Android Studio의 디버거를 사용할 수 있습니다.

## 2. 다시 USB 모드로 돌아가기

디버깅이 끝나고 다시 USB를 통해 ADB를 사용하고 싶다면 다음 명령어를 입력합니다.
```bash
adb usb
```

---

## 💡 프로젝트 팁

*   **기기 IP 주소 확인 방법**: 설정 > 휴대전화 정보 > 상태 > IP 주소에서 확인하거나, USB 연결 상태에서 `adb shell ip addr show wlan0` 명령어로 확인할 수 있습니다.
*   **안드로이드 11 이상 (무선 디버깅)**: 안드로이드 11 이상의 기기라면 '개발자 옵션' 내의 **'무선 디버깅'** 기능을 사용하여 QR 코드나 페어링 코드로 더 쉽게 무선 ADB를 설정할 수 있습니다.
*   **연결 유지**: 기기가 재부팅되거나 네트워크 상태가 바뀌면 연결이 끊어질 수 있습니다. 이 경우 다시 위 과정을 반복해야 합니다.
*   **AOA 장치 연결**: 이제 비워진 USB 포트에 AOA 액세서리 장치를 연결하고 테스트를 진행하면 됩니다.
